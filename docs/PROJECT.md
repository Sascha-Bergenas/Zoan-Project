## Projektbeskrivning

Vi ville skapa en app som inte bara är funktionell, utan också personlig och rolig att använda. Resultatet blev Zoan, en produktivitetsapp byggd i React och TypeScript med Supabase som backend. Målet var att hjälpa användaren att strukturera sin arbetsdag på ett smart sätt, utan att det ska kännas som ännu ett tråkigt verktyg. Vi ville att appen skulle kännas som något man faktiskt vill öppna.

Hjärtat i applikationen är timern, som låter användaren hålla koll på sina arbetspass med stöd för tre olika lägen: Deep Work, Möte och Chill. Tanken bakom lägena är att olika typer av arbete kräver olika fokus. Deep Work passar för långa koncentrerade pass, Möte för mer sociala och strukturerade situationer, och Chill för dagar när man kanske inte riktigt är på topp. Varje läge har anpassade rasttider som användaren själv kan ställa in via inställningssidan, och appen påminner automatiskt när det är dags att ta en paus. Timern håller också koll på den totala tiden sedan sessionen startades och visar det i en topbar längst upp på skärmen, tillsammans med information om nästa rast och nuvarande läge.

Att bygga timern var en av projektets större tekniska utmaningar. Vi behövde hantera pauser, ackumulerad tid och rastpåminnelser på ett bra sätt. Till slut löste vi det med en reducer-baserad arkitektur via useReducer och en dedikerad TimerContext som håller all timerlogik samlad på ett ställe. Det gjorde koden mycket lättare att resonera kring och felsöka.

För att göra upplevelsen mer personlig finns i dashboarden ett profil-kort, där användaren ser sitt användarnamn, sin profilbild och dagens datum. Profilbilden kan laddas upp och lagras i Supabase Storage. Som en liten extra touch visas också ett slumpmässigt roligt inspirationscitat varje gång man går in på sidan. Det är en liten detalj, men det ger appen lite personlighet!

Användaren kan uppdatera sitt användarnamn och sina rastinställningar direkt från settings-sidan, och ändringar sparas automatiskt i både localStorage och databasen så att inget försvinner mellan sessioner. Vi lade också till stöd för mörkt och ljust tema via ThemeContext, vilket gör appen bekvämare att använda oavsett ljusförhållanden och personliga preferenser. Det visar hur man kan använda Reacts Context API för att hantera globala UI-inställningar utan att det blir rörigt.

En annan lite mer "rolig" funktion är Smarta rekommendationer. Där kan användaren skriva hur de mår, och sedan får de ett förslag på arbetsläge och arbetstid via ett eget AI-drivet API. Om man skriver att man är trött kan appen till exempel rekommendera ett Chill-pass med kortare arbetstid, medan en pigg känsla kanske landar i Deep Work. Det var kul att få integrera AI i projektet på ett sätt som faktiskt tillför något för användaren, och det ger appen en modern touch.

På historik-sidan kan inloggade användare se sina tidigare sessioner hämtade från Supabase, och det finns möjlighet att manuellt lägga till sessioner via ett modal-fönster om man glömt att logga något. Sessionerna visas även i grafform för att ge en visuell bild av arbetsvanorna över tid, vilket gör det lättare att se mönster i hur man jobbar. För användare som inte är inloggade sparas data lokalt via localStorage som fallback, så att appen går att använda även utan konto.
Tekniskt sett använde vi Context API för att hantera global state för autentisering, tema och timer. Autentiseringen sköts via Supabase Auth med stöd för inloggning, registrering och utloggning. Vi skrev också enhetstester med Vitest och Testing Library eftersom det var ett krav i projektet. Testerna fokuserade på inställningssidan och täcker bland annat att spara användarnamn och hantera profilbildsuppladdning.

Sammantaget är Zoan ett projekt vi är stolta över. Vi har lärt oss mycket om hur man strukturerar en större React-applikation, hur man arbetar med en databas och autentisering, och inte minst hur man tar ett projekt från idé till något som faktiskt fungerar och känns genomtänkt. Det har varit utmanande ibland, men också roligt.
